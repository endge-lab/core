# AGENTS.md - AI Agent Documentation

This document provides a comprehensive overview of the Vue 3 starter template project for AI agents and developers.

## Project Overview

**Type**: Vue 3 Single Page Application (SPA) Starter Template

**Framework**: Vue 3 with TypeScript

**Build Tool**: Vite

**Package Manager**: pnpm

**Node Version**: >=22.12.0

This is a production-ready Vue 3 starter template with modern tooling, multi-branding support, internationalization, and a comprehensive component library.

## Core Architecture

### 1. Application Entry (`src/main.ts`)

The application initializes with:
- **Pinia**: State management
- **Vue Router**: Client-side routing
- **Vue i18n**: Internationalization
- **Branding Plugin**: Multi-brand theming
- **Dark/Light Mode**: Auto-detection via `@vueuse/core`

```typescript
import { createPinia } from 'pinia'
import { createApp } from 'vue'
import { i18n } from '@/i18n'
import { branding } from '@/lib/branding.ts'
import App from './App.vue'
import router from './router'

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.use(i18n)
app.use(branding)
app.mount('#app')
```

### 2. Layout System (`src/App.vue` + `src/components/layouts/`)

**Dynamic Layout Switching**: Routes can specify `meta.layout` to switch between layouts:
- `empty`: Minimal layout without chrome
- `default` (Main): Full layout with header, sidebar, breadcrumbs

**Error Boundary**: Global error capture with `onErrorCaptured` that displays errors without crashing the app (`Error.vue`).

**Layout State Management**: Global reactive state for:
- Breadcrumbs with configurable limit
- Loading indicators
- Sidebar state (expanded/collapsed)
- Fullscreen content dimensions
- Page title

**Key Composable**: `useLayout(options)` - Manages page title, breadcrumbs, and loading state reactively.

### 3. Multi-Branding System (`src/lib/branding.ts`)

**Features**:
- Brand-specific themes (CSS variables)
- Brand-specific favicons and logos
- Brand-specific i18n overrides

**Structure**:
```
src/assets/branding/
  ├── default/
  │   ├── manifest.json    # { name: "Brand Name", description: "..." }
  │   ├── icon.svg
  │   ├── theme.css        # CSS custom properties
  │   └── locale/
  │       ├── en.json      # Overrides for base en.json
  │       └── ru.json
  └── [other-brand]/
      └── ...
```

**Usage**:
```typescript
import { useBranding } from '@/lib/branding'

const { branding, brandings, currentBranding } = useBranding()
// branding.value = 'brand-name' to switch brands
```

**Environment Variable**: `VITE_BRANDING=default` sets the brand.

### 4. Internationalization (`src/i18n/index.ts`)

**Supported Locales**: English (`en`), Russian (`ru`)

**Features**:
- Type-safe translations (English is the schema)
- Brand-specific locale overrides (deep merge)
- Persisted locale selection in localStorage
- Reactive locale switching

**Brand Override Mechanism**:
1. Base translations loaded from `src/i18n/locales/*.json`
2. Brand-specific overrides loaded from `src/assets/branding/[brand]/locale/*.json`
3. Deep merge applied reactively when brand changes

**Usage**:
```typescript
import { useI18n } from 'vue-i18n'

const { t, locale } = useI18n()
locale.value = 'ru' // Switch language
```

### 5. Routing (`src/router/routes.ts`)

**Route Structure**:
- `/` - Home page
- `/deleteme/demo/*` - Demo pages (formatters, errors, toast, header, layout, questions)
- `/:pathMatch(.*)*` - 404 catch-all with Error view

**Route Meta**: Routes can define `meta.layout` to specify layout type.

### 6. State Management

**Pinia**: Installed and ready for use (no stores defined in template).

**Global Reactive State**:
- Layout state (breadcrumbs, loading, sidebar)
- Branding state (current brand, theme)
- i18n state (current locale)

### 7. Component Library

**UI Framework**: Custom components based on:
- **Reka UI**: Headless component primitives
- **shadcn-vue**: Component patterns (configured in `components.json`)
- **TailwindCSS 4**: Utility-first styling
- **Lucide**: Icon library
- **Vaul Vue**: Drawer component
- **Vue Sonner**: Toast notifications

**72 UI Components** in `src/components/ui/` including:
- Buttons, inputs, forms
- Dialogs, drawers, popovers
- Tables, cards, badges
- Navigation components
- And more...

## Build & Deployment

### Scripts

```bash
# Development
pnpm dev                 # Start dev server

# Production Build
pnpm build              # Type check + lint + build
pnpm build-only         # Build without checks
pnpm preview            # Preview production build

# Quality Checks
pnpm type-check         # TypeScript type checking
pnpm lint-check         # ESLint check
pnpm lint-fix           # ESLint auto-fix
```

### Docker Build

Multi-stage build process:
1. **Base**: Install pnpm and dependencies
2. **Dist**: Build application with `--mode=$BUILD_MODE`
3. **Nginx**: Serve static files with nginx-unprivileged

**Build Arguments**:
- `BUILD_MODE`: Environment mode (development, test, production)
- `ENDPOINT`: Base path for deployment

```bash
docker build --build-arg BUILD_MODE=production --build-arg ENDPOINT=/app -t app .
```

### Environment Variables

**Build-time** (Vite):
- `VITE_BRANDING`: Default brand name
- `VITE_DEFAULT_LOCALE`: Default locale (en/ru)
- `VITE_VERSION`: Injected from `package.json`
- `VITE_GIT_SHA`: Injected from git commit

All `VITE_*` variables are embedded at build time and accessible via `import.meta.env`.

## Utilities & Helpers

### `src/lib/utils.ts`

- **`cn(...inputs)`**: Merge Tailwind classes with conflict resolution
- **`deepClone<T>(obj)`**: Deep clone objects via JSON
- **`deepMerge<T>(base, override)`**: Deep merge objects (for i18n overrides)
- **`remToPx(value)`**: Convert rem to pixels

### `src/lib/duration.ts`

Duration formatting utilities using `@formatjs/intl-durationformat` and `iso8601-duration`.

### `src/components/layouts/main/layout.ts`

- **`useLayout(options)`**: Composable for page-level layout management
  - Set page title, breadcrumbs, loading state
  - Reactive and auto-cleanup on unmount
- **`getLayoutState()`**: Access global layout state
- **`useStateProvider(state)`**: Provide layout state from layout components
- **`resetLayout()`**: Reset layout state (for router hooks)

### `src/lib/branding.ts`

- **`useBranding()`**: Access branding system
  - `branding`: Current brand ref (writable)
  - `brandings`: List of available brands
  - `currentBranding`: Current brand metadata

## Component Patterns

### Using Layouts

Set layout in route meta:
```typescript
{
  path: '/page',
  component: () => import('@/views/Page.vue'),
  meta: { layout: 'empty' } // or 'default'
}
```

### Using Layout Composable

```typescript
import { useLayout } from '@/lib/layout'

const { sidebarState } = useLayout({
  title: 'Page Title',
  breadcrumbs: [
    { title: 'Home', href: '/' },
    { title: 'Current Page' }
  ],
  isLoading: false
})
```

### Using i18n

```typescript
import { useI18n } from 'vue-i18n'

const { t, locale } = useI18n()

// In template
{ { t('key.path') } }

// Switch language
locale.value = 'ru'
```

## Best Practices for AI Agents

### When Adding New Features

1. **Follow existing patterns**: Use Composition API with `<script setup>`
2. **Type everything**: Leverage TypeScript for safety
3. **Use composables**: Extract reusable non-business logic to `src/lib/`
4. **Respect the layout system**: Use `useLayout()` for layout state
5. **Support i18n**: Add translations to `src/i18n/locales/*.json` for every available language
6. **Use UI components**: Leverage existing components in `src/components/ui/`
7. **Use shadcn-vue**: When you need new components check shadcn-vue component registry and install them with `pnpm dlx shadcn-vue@latest add <component name>` or by following instructions in the docs

### When Modifying Code

1. **Preserve branding system**: Don't break multi-brand support
2. **Maintain i18n**: Keep translations in sync
3. **Update types**: Keep TypeScript types accurate
4. **Maintain code-style**: Ensure code-style using ESLint

### When Adding Dependencies

1. **Use pnpm**: `pnpm add <package>`
2. **Check compatibility**: Ensure Node 20.19+ or 22.12+ support
3. **Update lockfile**: Commit `pnpm-lock.yaml`
4. **Consider bundle size**: Keep production bundle lean

### When Creating Components

1. **Use TypeScript**: Define prop types with `defineProps<T>()` and v-models with `const model = defineModel('model', { required: true })`
2. **Use Composition API**: Prefer `<script setup lang="ts">`
3. **Style with Tailwind**: Use utility classes
4. **Make it reusable**: Consider adding to `src/components/ui/`
5. **Support i18n**: Use `useI18n()` for user-facing text
